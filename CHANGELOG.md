# 更新日志

本文档记录虚拟人生插件的主要功能、行为变化与修复。

格式参考 Keep a Changelog，版本号以 `metadata.yaml` 的历史记录为准。

## [v1.10.2] - 2026-07-23

### 变更

- `underpants` 现在与其他必选穿搭分类使用同一套完整性校验；缺少该分类的旧日程不再兼容。
- 移除重复的穿搭补充系统提示词配置，内裤分类要求仅由穿搭系统提示统一声明。
- 完整日程改用单一共享提示模板，不再重复发送人格、日期和创意上下文。
- 历史日程仅保留最近一天的完整时间线，较早日期压缩为主题、心情、穿搭风格和活动摘要。
- 重写日程不再把当前时间线同时作为历史详情重复发送，局部模板也移除了不可达的完整生成说明。
- 所有生成提示词与模板移入独立的 `prompt_settings` 配置组，并在配置界面默认折叠。

## [v1.10.1] - 2026-07-22

### 修复

- 穿搭重写兼容模型直接返回 `style`、`summary`、`items` 的情况，并继续执行完整穿搭校验。
- 生成失败重试会携带最近一次模型输出与精确校验原因，不再累积所有历史失败提示。
- 新生成和重写的穿搭必须包含独立的 `underpants` 内裤分类；旧日程缺少该分类时仍可正常读取。

### 变更

- 新增可配置的 `generation_retry_prompt_template`，默认额外重试次数由 `1` 提升为 `2`。
- 日程与穿搭系统提示词增加明确的单顶层 JSON 骨架，禁止嵌套到 `data`、`result` 等包装字段。
- 新增穿搭补充系统提示词配置，确保已有自定义提示词也会收到 `underpants` 必要分类规则。

## [v1.10.0] - 2026-07-21

### 新增

- 新增 `/虚拟日程 重写日程 [补充要求]`，保留主题、心情和穿搭，仅重写时间线、主动窗口与消息预算。
- 新增 `/虚拟日程 重写穿搭 [补充要求]`，保留主题、心情和时间日程，仅重写穿搭；默认排除当前风格，并支持通过补充要求指定风格池中的风格。

### 变更

- 原 `/虚拟日程 重写 [补充要求]` 保留为完整重写命令。
- 局部重写只有在新内容通过完整校验后才会覆盖原计划，失败时保留原内容。
- 生成提示词拆分为日程和穿搭两组；完整生成合并两组提示词后单次调用，局部重写只调用对应提示词组。
- 旧版 `generation_system_prompt` 和 `prompt_template` 配置项不再读取，新配置项会由 AstrBot 自动补入。

## [v1.9.2] - 2026-07-19

### 变更

- 每日日程生成时间默认值改为 `00:00` 。
- 主动消息窗口生成时，会将 `intent` 字段约束为关联活动的角色表达。
- 增加了一些默认日程池选项。

## [v1.9.1] - 2026-07-16

### 修复

- 主动消息发送成功后写入当前 AstrBot 对话历史，使后续对话能够感知已经发送的主动内容。
- 当前会话没有对话时自动创建对话，并以系统触发说明与主动消息正文组成标准 user-assistant 消息对。

## [v1.9.0] - 2026-07-16

### 新增

- 新增 `/主动消息 执行时间`，展示当前会话所有窗口、延迟窗口、沉默、睡眠异常和回访任务的具体执行时间。

### 变更

- 主动消息状态、立即发送、回访列表和取消回访统一迁入 `/主动消息` 命令组。

## [v1.8.5] - 2026-07-16

### 变更

- 普通主动消息根据日程可打扰程度按 `0%/25%/70%/100%` 的默认概率触发。
- 日程窗口未命中概率或遇到睡眠时，会延迟到下一可打扰时段，并在消息意图中注明原定日程已结束。

## [v1.8.4] - 2026-07-16

### 修复

- 主动消息窗口现在必须引用存在的来源日程项，且窗口时间必须位于来源项目的时间范围内。

## [v1.8.3] - 2026-07-16

### 新增

- `/虚拟日程 查看` 展示今日主题、心情状态、当前生效的大时间表阶段、今日特殊时间段与今日节日。
- 新增独立 PNG 图片预览脚本，可按参数生成日程时间轴、穿搭、阶段列表和阶段详情。
- 预览脚本内置完整示例数据，支持深浅主题、图片宽度、字体、输出目录和浏览器路径参数。
- 预览脚本复用生产环境的 `ScheduleImageRenderer`，通过 Jinja2 与 Playwright 直接输出图片，无需启动 AstrBot。

### 变更

- 时间轴图片原当前活动摘要卡改为当前大时间表阶段；右上角继续显示当前活动时段。
- 今日特殊时间段仅展示当前日期正在生效的项目及约束，节日仅展示当天节日。
- 没有当前大时间表阶段时显示“暂无当前阶段”，没有特殊时间段或节日时隐藏对应区域。

## [v1.8.2] - 2026-07-16

### 新增

- 新增 `/虚拟日程 穿搭` 命令，独立渲染今日穿搭图片。
- 为结构化穿搭新增持久化的 `outfit.style` 字段，保存创意池选中的穿搭风格。
- 穿搭视图展示穿搭风格、今日主题、今日心情、造型概述和单品明细。
- 时间轴视图增加当前日期所处时间段、活动、地点、状态与可联系程度摘要。
- 为时间轴查询和穿搭查询分别提供文字回退格式。

### 变更

- `/虚拟日程 查看` 不再混合展示穿搭，仅渲染 24 小时时间轴和主动消息窗口。
- 日程生成提示词要求 `outfit` 包含 `style`、`summary` 和 `items`。
- 历史日程摘要加入穿搭风格，帮助后续日程减少重复造型。

### 兼容

- 旧结构化日程缺少 `outfit.style` 时继续正常读取，并显示为“未记录”。
- 保留 `render_daily` 兼容入口，其行为转发到新的时间轴渲染器。

## [v1.8.1] - 2026-07-15

### 修复

- 日程关键词命中时，确保注入当前时间、活动、地点、状态和可打扰程度等基础状态。
- 避免基础模块与日程模块同时启用时重复注入相同状态。
- 智能注入日志增加实际注入内容，便于排查模块匹配和字符截断问题。

## [v1.8.0] - 2026-07-15

### 新增

- 新增 `smart_context_injection.base_module_enable`，允许控制是否在每次普通聊天中始终注入基础状态。
- 智能注入关键词、模块开关和限制改为直接读取插件配置。

### 变更

- 默认关闭持续基础状态注入，仅在关键词命中时加入相关模块，减少普通聊天上下文占用。
- 配置中的关键词列表为空时，对应注入模块不再触发。
- 移除插件自带的 `/sid` 命令，避免与 AstrBot 或其他插件提供的同名命令冲突。

## [v1.7.0] - 2026-07-15

### 新增

- 新增主动消息分段器，可将短消息按中英文句末标点、换行或自定义关键词拆分发送。
- 支持 `words` 与 `regex` 两种分段模式。
- 支持分段后的正则内容清理。
- 支持随机间隔和按中英文字数计算的对数间隔。
- 增加不分段字数阈值，长文本可保持整条发送。

### 变更

- 将主动消息分段配置统一收敛至 `delivery_settings.segmented_reply_settings`。
- 调整分段配置命名、条件显示和默认规则，使配置结构与实际发送行为一致。

## [v1.6.0] - 2026-07-15

### 新增

- 新增 `/虚拟人生 订阅会话` 管理员命令。
- 自动识别当前会话是私聊还是群聊，并写入对应 UMO 白名单。
- 订阅后立即生成或读取今日日程、安排主动消息并保存会话状态。
- 重复订阅时刷新当前会话的主动消息调度。

## [v1.5.0] - 2026-07-15

### 新增

- 大时间表自动叠加中国法定节假日。
- 支持春节、元宵、端午、七夕、中秋、重阳、腊八和除夕等传统节日。
- 将当天及临近节日信息提供给每日日程生成器。
- 日程主动窗口支持确定性随机偏移，默认在计划时间前后 15 分钟内变化。

### 变更

- 主动窗口的随机偏移不会超出其关联活动时段。
- 用户明确指定时间的一次性回访不参与随机偏移。
- 节日信息加入长期阶段图片和日程规划上下文。

## [v1.4.0] - 2026-07-15

### 新增

- 新增本地关键词驱动的智能虚拟状态注入，不额外调用 LLM。
- 支持基础状态、外显穿搭、内衣与打底、今日日程、大时间表等按需注入模块。
- 完整日程或完整大时间表查询会提示模型优先调用对应 LLM Tool，避免根据摘要虚构内容。
- 增加注入字符上限、里程碑未来天数和各模块关键词配置。
- README 增加智能注入与日程生成的 Token 预算说明。

### 变更

- 插件名称由“主动虚拟日程”更名为“虚拟人生”。
- 插件标识与仓库地址改为 `astrbot_plugin_virtual_life`。
- 重整配置描述、分组和提示信息，提高 AstrBot 配置页面可读性。

## [v1.3.0] - 2026-07-15

### 新增

- 将每日穿搭升级为结构化对象，包含整体描述和分类单品。
- 支持发型、头饰、内衣、上装、下装、连衣裙、腿部穿搭、外套、鞋履、配饰、包具、妆容、香氛和其他造型分类。
- 日程图片增加穿搭概述和单品卡片。
- 大时间表阶段图片增加返回列表和查看指定阶段的导航提示。

### 变更

- 强制校验穿搭的必要分类和主体服装组合。
- 旧版字符串穿搭不再视为有效的结构化日程，会触发重新生成。
- 改进大时间表命令在缺少阶段 ID 时的帮助与导航行为。

## [v1.2.0] - 2026-07-14

### 新增

- 新增基于 HTML 模板的日程图片渲染器。
- 每日日程支持 24 小时时间轴、当前活动高亮、状态和可打扰程度展示。
- 大时间表支持阶段列表、阶段详情和待批准草稿图片。
- 支持深色与浅色图片主题、图片宽度和字体配置。
- 图片渲染失败时自动回退为文字结果。

### 修复

- 规范长期阶段优先级为数值并统一归一化处理。
- 校验日程与长期固定事件中的空时间字段。
- 加强长期阶段生成规则、时间格式和结构校验。
- 改善中文提示与错误文本的可读性。

## [v1.1.0] - 2026-07-14

### 新增

- 新增按人格隔离的大时间表系统，可描述学期、假期、考试周、项目周期、冲刺和发布阶段。
- 支持周期性固定安排、特殊日期、特殊时期、里程碑、地点、参与人、约束和阶段优先级。
- 支持长期阶段草稿的生成、导入、查看、批准、拒绝和重新生成。
- 首次大时间表必须由管理员批准后生效。
- 当前最后阶段临近结束时自动生成连续后续阶段，并通知管理员批准。
- 每日日程生成会读取当前有效阶段，并保留当天必须发生的固定事件。
- 将每日日程和大时间表命令整理为独立命令组。

### 变更

- 每个人格独立保存已批准阶段、待批准草稿和管理员通知会话。
- 重叠阶段按优先级、范围长度和录入顺序选择。
- 无法消解的固定事件冲突会阻止日程生成，而不是生成不一致时间线。

## [v1.0.0] - 2026-07-14

### 首次发布

- 按当前生效人格生成结构化的每日虚拟日程，同一人格可在多个会话共享日程。
- 日程包含主题、心情、穿搭、全天活动时间线、状态、可打扰程度和主动分享窗口。
- 私聊与群聊分别维护 UMO 白名单，不向未知会话主动发送消息。
- 按会话生成每日主动消息预算，并允许 LLM 在管理员硬上限内建议小幅增加。
- 忙碌、专注和睡眠状态可抑制或延后主动消息。
- 连续主动消息无人回应达到阈值后暂停，收到用户新消息后恢复。
- 睡眠期间支持低概率梦醒、起夜或失眠式主动消息。
- 支持创建、查询、取消和持久化一次性主动回访任务。
- 提供完整日程查询、主动回访管理等 LLM Tools。
- 支持管理员查看状态、立即触发主动消息和重写日程。
- 日程生成失败时自动重试，并加强全天连续覆盖、时间格式和 JSON 结构约束。
- 支持为日程生成和主动消息分别选择 LLM Provider。
- 支持参考同一人格的近期历史日程，保持生活连续性并减少重复安排。
- 使用 JSON 文件持久化人格日程、会话状态和回访任务，并通过异步锁与临时文件替换保证写入安全。
